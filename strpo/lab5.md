### Исследование альтернатив

Для управления похожим процессом можно использовать Jira и Redmine.

Jira поддерживает доски задач, настраиваемые статусы, связи между задачами, кастомные поля и интеграцию с GitHub. При подключении GitHub в Jira можно видеть связанные ветки, коммиты и pr`ы, если в них используется ключ задачи. У Jira есть разные тарифы, где бесплатный план подходит для небольших команд, но имеет ограничения по автоматизациям и расширенным возможностям администрирования.

Redmine - open-source система управления проектами с самостоятельным развёртыванием. Поддерживает задачи, собственные статусы, роли и права, кастомные поля, диаграмму Ганта, календарь, wiki, дорожные карты и работу с несколькими проектами. Основное отличие от Jira в том, что сам Redmine распространяется бесплатно, но его нужно разворачивать и администрировать самостоятельно. 

### Выбранная альтернатива: Jira

Похожий процесс можно было бы реализовать в Jira следующим образом:

- создать проект для курсовой работы;
- завести те же задачи;
- настроить статусы;
- добавить кастомные поля: плановая дата начала, плановая дата окончания, тип работы;
- связать задачи через зависимости;
- для задачи “Анализ задания курсовой работы” провести согласование через pull request, связанный с задачей Jira по ключу в ветке, коммите или названии PR.
